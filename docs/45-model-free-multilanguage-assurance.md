# Model-free multi-language assurance

Sergeant's permanent officers must remain useful when no model route is available. Models amplify the council's reasoning; they do not create the council or replace its evidence duties.

The model-free language matrix is a blind, project-shaped benchmark. Expected findings are kept outside each temporary review workspace until the review returns. Every planted defect has a clean counterpart using the same language and project shape.

## Current matrix

| Language | Defect obligation | Clean-control obligation |
|---|---|---|
| TypeScript | Request input reaches shell command execution | Fixed executable and bounded arguments remain quiet |
| Go | Request value is concatenated into SQL | Bound SQL parameter remains quiet |
| Rust | Request path reaches file access without containment | Canonicalized, contained path remains quiet |
| Java | Privileged route lacks authorization | Explicit authorization guard remains quiet |
| C# | Concurrent work mutates shared state | Atomic increment remains quiet |
| Ruby | Nested iteration creates scaling risk | Linear iteration remains quiet |

The benchmark requires all twelve verdicts to be correct, all nine expected findings to match, zero false positives, zero false negatives, and zero model calls. Minor performance and concurrency findings remain advisory and cannot force `REQUEST_CHANGES`.

## Permanent officer ownership

- Scout identifies language, manifest, tests, CI, documentation, and changed scope.
- Medic owns command injection, SQL taint, path containment, and authorization evidence.
- Mechanic owns concurrency and performance evidence.
- Judge admits actionable findings, preserves advisories, and recomputes the final verdict after adjudication.
- Quartermaster records that the deterministic formation consumed no model capacity.

Language-aware rules are guard-paired. A sink term alone is not enough: parameter binding, containment guards, authorization annotations, and atomic or locking primitives are explicit falsifiers. This is what makes the formation useful without a model while leaving room for connected models to improve reasoning over unfamiliar control flow, framework semantics, and ambiguous architecture.

## Required proof

The `Review Intelligence Proof` workflow runs:

1. focused language and policy tests;
2. the original blind review benchmark;
3. the model-free multi-language benchmark from the source tree;
4. both benchmarks again from a freshly built and installed wheel;
5. credential scanning over the resulting artifacts.

An installed package that omits the multi-language cases fails the workflow. A future rule that restores a false positive in any clean control also fails the workflow.
