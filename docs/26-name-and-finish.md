# Name and Finish

## Final project name

**Sentinel Review**

## Why this name

Sentinel means a guard that watches, checks, and protects before damage reaches users.

That matches the intended reviewer:

- it stands inside the GitHub workflow
- it reviews independently
- it challenges its own conclusions
- it uses the THETECHGUY engineering standard
- it learns from accepted reviewer/human feedback
- it treats external reviewers as optional teachers, not authorities

## Product identity

Public name: **Sentinel Review**

Internal THETECHGUY/Hunter name: **Main Review**

Suggested repository name: `sentinel-review`

## Finished scope for v1 foundation

The v1 foundation is considered finished when these are present and proven:

- independent PR reviewer
- GitHub workflow entrypoint
- proof gates
- challenge mode
- decision workspace
- consensus
- memory learning
- memory verification
- clean-clone CI proof

## What remains after v1

Future improvements are depth upgrades, not foundation blockers:

- deeper language-specific analyzers
- richer cross-file reasoning
- direct GitHub review posting through a hardened app/service
- Hunter-backed persistent memory
