# Sergeant 0.5.0 — Model-Free Core and Governed Learning

Sergeant 0.5.0 is the largest reviewer update since the 0.4 semantic-routing release. It strengthens the deterministic reviewer, permanent-officer formation, proof integrity, self-hosted operation, external-review comparison, and controlled learning pipeline while keeping Sergeant—not a model provider—as final authority.

## Model-free reviewer core

- Expanded deterministic review across security, authorization, lifecycle, concurrency, data integrity, path handling, protocol, state-transition, and cross-language failure mechanisms.
- Preserved immutable blind scores separately from learned closure.
- Proved three consecutive untouched-transfer rounds with **9/9 fresh blind detections** across OCaml, Julia, Crystal, Zig, R, Erlang, C++, Java, and Python after the transfer-31 reviewer freeze.
- Kept normal review operational with model calls disabled.

## Permanent officers and Cpl governance

- Added the deterministic permanent-officer formation and kept specialist ownership separate from optional model support.
- Added elastic Cpl council rounds, bounded recruitment, evidence rebriefs, and explicit adjudication states.
- Added a canonical finding ledger that separates confirmed findings, advisories, rejected claims, unresolved assurance, and required actions.
- Prevented repeated model wording from multiplying one root defect into several release blockers.
- Added quota-aware provider failover, credential-safe diagnostics, mission qualification, and Cloudflare Workers AI compatibility without making Cloudflare mandatory.

## Reviewer intelligence and comparison

- Added blind reviewer-quality benchmarks with answer leakage blocked.
- Added evidence-first Sergeant-versus-external-review comparison with exact-head binding and stale-comment exclusion.
- Added append-only verified experience and recurrence tracking while keeping current repository evidence authoritative.
- Added model-free campaign and transfer evidence suitable for independent replay.

## Production and self-hosted operation

- Added the standalone `sergeant-serve` service with bounded requests, optional bearer authentication, rate limiting, one active mission, and read-only workspace behavior.
- Added the browser-hosted Command Center outside the IDE.
- Added signed GitHub webhook intake with replay protection and no automatic review, posting, merge, or repository-write authority.
- Added hardened container and installed-wheel proof.
- Strengthened live GitHub ingestion with host validation, bounded pagination, path containment, redaction, and GET-only evidence.

## Proof-integrity repairs

- Added complete GitHub pull-request file pagination and fail-closed handling at the API boundary.
- Made fresh untouched-transfer provenance mandatory by classification rather than manifest opt-in.
- Strengthened selector truth propagation, exact sparse checkout, pure JSON summaries, and malformed-row regression coverage.
- Preserved historical negative attempts and original frozen scores rather than rewriting them after repairs.

## Controlled self-learning preview

- Added an adaptive multilingual curriculum that increases repository difficulty only after complete, low-noise evidence windows.
- Preserved the existing ten-times private-force law, scaling bounded investigations from 20 to 120 workers according to difficulty.
- Added language-family rotation and capped Rust at two cases per ten so systems-language learning cannot dominate the curriculum.
- Added isolated Teacher, Prosecutor, and Defender contracts with GitHub Models or separate Hermes-compatible endpoints as optional learning transports.
- Added a governed queue and draft proposal export.
- **No lesson is automatically promoted and no proposal is automatically merged.** Every accepted lesson still requires executable negative controls and unrelated-language transfer.

The first controlled council round completed safely with three multilingual cases, zero worker transport errors, and zero automatic promotions or merges. The Defender rejected all three candidate lessons because the evidence was insufficient or overfit-prone. This is recorded as successful fail-closed behavior, not as three learned rules.

## Safety boundary

Sergeant 0.5.0 does not grant models, learning workers, webhooks, or external reviewers final verdict, repository-write, patch-application, or merge authority. Normal review remains usable without an AI login or continuously running large GPU. Optional models may add evidence or accelerate lesson proposals, but admitted findings and permanent lessons remain governed by Sergeant's deterministic evidence contracts.

## Release proof required

Publication is allowed only after the full stacked implementation reaches `main` and the exact release head passes:

- CI and clean-clone proof;
- Main Review with actual-verdict enforcement;
- multiplatform Python, VS Code, Command Center, and JetBrains proof;
- model-free campaign validation and final static holdout;
- standalone service and live GitHub ingestion proof;
- coordinated version and package checks;
- credential scanning;
- CodeRabbit disposition with no unresolved actionable finding.

This release does not claim universal defect detection or autonomous unrestricted self-training. It ships an evidence-bound reviewer and a controlled learning factory whose weak lessons fail closed.
