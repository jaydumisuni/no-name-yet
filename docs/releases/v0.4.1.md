# Sergeant 0.4.1 — Useful Model-Free Baseline

Sergeant 0.4.1 publishes the substantial, already-merged engineering work that accumulated after the original 0.4.0 tag. It is an interim usability release while the governed 0.5.0 learning stack completes its controlled validation period.

## Useful without an AI account

- Deterministic repository, pull-request, changed-file, current-file, architecture, security, lifecycle, concurrency, data-integrity, and regression-risk review remains available with Cpl disabled.
- Permanent specialist officers can investigate correctness, security, architecture, tests/contracts, performance, and concurrency without requiring a model route.
- `sergeant review`, `sergeant pr-review`, proof-suite, final-proof, battle-test, and IDE Bench commands are packaged in the Python CLI.
- Missing optional model routes do not disable normal deterministic review.

## Standalone and IDE operation

- Adds the `sergeant-serve` standalone service with bounded requests, optional bearer authentication, rate limiting, one active mission, and read-only workspace behavior.
- Adds the browser-hosted Command Center outside the IDE.
- Ships the full VS Code Command Center and the JetBrains preview with one-active-mission protection, evidence views, and report export.
- Packages the shared Command Center resources in the Python distribution.

## GitHub and reviewer evidence

- Strengthens read-only GitHub pull-request ingestion, pagination, path containment, redaction, and failure reporting.
- Adds reviewer benchmarks and `sergeant-compare` for transparent external-review comparison.
- Adds battle fixtures, static review-signal comparison, and clean-clone proof.
- Keeps live fetch and external-review evidence separate from Sergeant's final authority.

## Optional Cpl and provider support

- Supports the Sergeant Cpl gateway, Ollama, LM Studio, explicit OpenAI-compatible endpoints, and Cloudflare Workers AI routing.
- Adds bounded council rounds, specialist assignments, provider failover, credential-safe diagnostics, and structured-result compatibility.
- Keeps model findings subordinate to repository evidence, deterministic tests, explicit contracts, and the Commander verdict.

## Safety boundary

Sergeant 0.4.1 does not execute pull-request-controlled shell commands, automatically modify reviewed repositories, apply patches, merge changes, expose model credentials, or treat a model/external reviewer as final authority.

## Why this patch exists

The original public 0.4.0 package did not contain most of the useful work now present on `main`. This patch publishes that proven baseline rather than leaving users on the old package until 0.5.0 finishes its controlled learning week.

Sergeant 0.4.1 does not include the adaptive self-learning queue, automatic curriculum escalation, or governed lesson-proposal system reserved for 0.5.0.
