# Controlled self-learning week 1

This phase converts Sergeant's proven manual curriculum into a governed automated loop while keeping normal review model-free.

## Round sequence

1. Collect merged bug-fix lineages from a multilingual GitHub pool.
2. Require complete PR-file pagination, production source, changed tests, bidirectional executable change, ancestry, and capability-addition rejection.
3. Freeze the exact Sergeant head before candidate collection.
4. Check out the defective revision and run Sergeant with models and workspace execution disabled.
5. Persist the blind result before revealing the fixing diff.
6. Send a bounded truth packet to three isolated workers:
   - **Teacher** derives a generalized mechanism, detector proposal, positive tests, negative controls, and transfer languages.
   - **Prosecutor** builds the strongest evidence-bound defect case and rejects competing explanations.
   - **Defender** searches for clean counterexamples, overfitting, missing evidence, and false-positive risk.
7. Save council-complete proposals to a separate draft GitHub PR.
8. Keep promotion blocked until executable negative controls and an unrelated-language transfer both pass.

## Worker transport

The preferred deployment is three separate Hermes Agent profiles exposed through their OpenAI-compatible API servers. Each profile has a separate URL, key, model identity, memory, skills, and state.

The initial GitHub-hosted week can use GitHub Models through the repository's included `models: read` permission. This is a temporary inference transport, not part of Sergeant's permanent review path. Switching `SERGEANT_LEARNING_BACKEND` to `hermes` routes the same role contracts to:

- `SERGEANT_HERMES_TEACHER_URL`
- `SERGEANT_HERMES_PROSECUTOR_URL`
- `SERGEANT_HERMES_DEFENDER_URL`

with separate repository secrets for each key.

No worker receives merge authority, shell tools, repository credentials, or permission to alter Sergeant directly.

## Difficulty and language law

Three complete low-noise rounds are required before the next repository tier. Promotion moves one tier only and fails closed when no harder candidate exists.

The curriculum forbids consecutive use of the same language, limits one family to two of the recent five cases, and limits Rust to two of the recent ten. Rust remains a useful accelerator for ownership, concurrency, lifecycle, error-path, and unsafe-boundary mechanisms, but every broad lesson must transfer outside Rust.

## GitHub authority boundary

Automated work may:

- collect and freeze cases;
- run blind reviews;
- create lesson proposals;
- save evidence as GitHub artifacts;
- open a draft proposal PR.

Automated work may not:

- rewrite a frozen blind score;
- auto-promote a lesson;
- auto-merge a proposal;
- expose fixing truth before blind freeze;
- weaken provenance, controls, or transfer requirements.
